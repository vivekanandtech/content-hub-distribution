Engineering

Challenges
Solved

Real problems encountered and solved during the build. Every challenge required creative problem-solving.

Challenge 01
Router Merge Problem
Make.com's Router cannot merge multiple routes into a single downstream module. Solved by redesigning the architecture so each route publishes independently to its own platform, eliminating the need to merge.
Challenge 02
JSON Parsing from AI Response
Mistral AI returns content embedded in a complex nested array structure with newline characters breaking JSON parsing. Solved using Make.com's replace() function with the correct array indexing: choices[1].message.content.
Challenge 03
Special Characters Breaking JSON
AI-generated content contains quotes, newlines, and special characters that break JSON when sent as HTTP body. Solved by switching from raw JSON body to form URL-encoded format, which handles escaping automatically.
Challenge 04
Zapier Data Parsing
Zapier receives form-encoded data as a single "Type" field string rather than parsed fields. Solved using Python's urllib.parse.parse_qs() function to decode the URL-encoded string into individual accessible fields.
Challenge 05
Twitter Thread Creation
Twitter/X native module was removed from Make.com. Solved using a Zapier webhook approach where Mistral AI generates 4 numbered tweets separated by a custom delimiter, which Zapier then splits and posts as a threaded conversation.
Challenge 06
Hugging Face API Deprecation
The original Hugging Face api-inference endpoint was deprecated mid-build. Pivoted to Mistral AI which provided superior JSON formatting, better instruction following, and a generous free tier of 1M tokens/month.
